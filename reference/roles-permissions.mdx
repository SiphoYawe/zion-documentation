Zion has five built-in roles: administrator, supervisor, monitor, student, and parent. Every account belongs to exactly one of these, and each role maps to a Clerk organization role. &quot;Teaching staff&quot; is the label Zion uses in the interface for the group of administrator, supervisor, and monitor accounts, not a separate role. There is no &quot;owner&quot; role inside a school&apos;s organization; the console that manages Zion&apos;s own tenant organizations is restricted to Zion&apos;s platform operators and sits outside any school&apos;s role system.

On top of the five built-in roles, administrators can create custom roles that grant additional permission keys to specific users. Custom roles are additive: they never replace a user&apos;s built-in role, they only extend what that person can do.

## The five roles

| Role | Typical user | Scope |
|------|-------------|-------|
| Administrator | Principal, head of school, academic director | Every learning centre and every feature in the organization |
| Supervisor | Learning centre teacher, lead teacher | Assigned learning centres only |
| Monitor | Teaching assistant, part-time or volunteer staff | Assigned learning centres, data entry focused |
| Student | Enrolled student | Own data only, through the student portal |
| Parent | Parent or guardian | Their own child&apos;s data only, through the parent portal |

<Note>
The student portal and parent portal are both shipped features. Students and parents sign in and see their own (or their child&apos;s) goals, progress, and PACE assignments; they are not future or in-development capabilities.
</Note>

## Permissions matrix

This matrix summarizes what each role can do across the features documented in this guide. &quot;Assigned centres&quot; means the user only sees data for learning centres they have been assigned to.

### Organization and staff

| Capability | Administrator | Supervisor | Monitor | Student | Parent |
|---|---|---|---|---|---|
| Edit school profile, terms, timezone | Yes | No | No | No | No |
| Create, edit, archive learning centres | Yes | No | No | No | No |
| View learning centres | All | Assigned only | Assigned only | No | No |
| Invite, edit, deactivate staff | Yes | No | No | No | No |
| Assign supervisors or monitors to a centre | Yes | No | No | No | No |
| Create and manage custom roles | Yes (`roles:manage`) | No | No | No | No |

### Students and PACEs

| Capability | Administrator | Supervisor | Monitor | Student | Parent |
|---|---|---|---|---|---|
| Add, bulk-import, deactivate students | Yes | No | No | No | No |
| Edit student profiles | Yes | Limited | No | No | No |
| View student rosters | All centres | Assigned centres | Assigned centres | No | No |
| Assign PACEs to students | Yes | Assigned centres | No | No | No |
| View own student or child data | All | Assigned centres | Assigned centres | Own only | Own child only |

### Goal Check, grading, and attendance

| Capability | Administrator | Supervisor | Monitor | Student | Parent |
|---|---|---|---|---|---|
| Set goals, mark completion | All centres | Assigned centres | Assigned centres | No | No |
| View Goal Check history | All | Assigned centres | Assigned centres | Own only | Own child only |
| Enter and submit grades | All centres | Assigned centres | No | No | No |
| Moderate grades, set thresholds | Yes | No | No | No | No |
| Mark and view attendance | All centres | Assigned centres | Assigned centres | No | No |

### Calendar, reports, and inventory

| Capability | Administrator | Supervisor | Monitor | Student | Parent |
|---|---|---|---|---|---|
| View calendar | Yes | Yes | Yes | Yes | Yes |
| Create centre events | Yes | Yes | No | No | No |
| Create organization-wide events, delete any event | Yes | No | No | No | No |
| Generate and view reports | All centres | Assigned centres | No | No | No |
| View cross-centre reports, share by email | Yes | No | No | No | No |
| View and check out inventory items | Yes | Yes (view, checkout) | No | No | No |
| Manage inventory (receiving, adjustments) | Yes | No | No | No | No |

<Note>
Some permission keys in the catalog below, such as viewing all centres&apos; goal check data or exporting it, cross-centre grading views, cross-centre attendance views, and cross-centre reports, are reserved for administrators today. Where the role editor shows a permission as not yet enforced, it is marked &quot;Coming soon&quot; rather than offered as a working grant.
</Note>

## Role details

### Administrator

Administrators have full access to every learning centre, every student, and every feature, including school settings, staff management, custom roles, and weekly school-wide reports. A school typically has one to three administrators.

### Supervisor

Supervisors manage the learning centres they are assigned to: setting goals, marking Goal Check progress, entering grades, marking attendance, generating centre reports, and viewing their centre&apos;s dashboard. They cannot create centres, invite staff, or see data from centres they are not assigned to.

### Monitor

Monitors enter Goal Check data (setting goals, marking progress) for their assigned centres. Their interface is deliberately narrower than a supervisor&apos;s: no dashboards, no report generation, no cross-centre visibility. Monitors are well suited to teaching assistants or volunteer staff who only need to record data.

### Student

Students sign in to the student portal and see their own daily goals, progress, and PACE assignments. Access is read-only and limited to their own record.

### Parent

Parents sign in to the parent portal and see their child&apos;s goals, progress, and PACE assignments, and can book meetings with their child&apos;s teaching staff. Access is read-only and limited to their own child&apos;s record. Parents are invited by the school; an administrator or supervisor can track invitation status and re-invite parents who have not yet connected.

## Custom roles

Administrators can layer custom roles on top of a user&apos;s built-in role from **Settings &gt; Roles**. A custom role is a named set of permission keys that gets added to whatever the user already has through their built-in role.

<Steps>
  <Step title="Open Settings > Roles">
    This page requires the `roles:view` permission, which administrators hold by default.
  </Step>
  <Step title="Create a role">
    Click **New role**, name it, add a description, and check the permission keys to grant from the catalog below.
  </Step>
  <Step title="Assign the role to a user">
    On a user&apos;s detail page under **Assigned Roles**, add the custom role. It applies on top of their built-in Clerk role, it does not replace it.
  </Step>
  <Step title="Review effective permissions">
    The user&apos;s **Effective Permissions** card shows the resolved union of their built-in role and any assigned custom roles, with the source of each grant.
  </Step>
</Steps>

Every built-in role is also mirrored as a system role in the same list, so administrators can see exactly which permission keys make up the administrator, supervisor, monitor, student, and parent bundles. System roles are marked with a System badge: they cannot be renamed or deleted, but a system role can be cloned to create a customizable copy. Deleting a custom role that is still held by users shows how many users hold it before you confirm.

Every change to a role or its permissions is written to a role change history, filterable by the person who made the change, from **Settings &gt; Roles &gt; View audit log**.

## Permission catalog

Permission keys are grouped by feature area. Administrators hold every key by default; supervisor and monitor bundles hold a smaller, centre-scoped subset; student and parent bundles are limited to viewing their own data and the calendar.

| Group | Permission keys |
|---|---|
| Students | view, create, edit, delete, view all centres, assign PACEs |
| Goal Check | view, set goals, mark completion, view all, export |
| Grading | view, submit, moderate, view all |
| Attendance | mark, view, view all |
| Calendar | view, create centre event, create organization event, delete any event |
| Reports | view, generate, view all, share |
| Settings | view, edit school profile, manage users, manage terms, manage centres |
| Roles &amp; Permissions | view roles, manage roles |
| Inventory | view, manage, checkout |

<Warning>
Not every permission key in the catalog is wired to an active gate yet. Keys such as cross-centre goal check export, cross-centre grading and attendance views, and cross-centre reports view are declared in the catalog for future use and appear as disabled, &quot;Coming soon&quot; options in the role editor rather than functioning grants.
</Warning>
