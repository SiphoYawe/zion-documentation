This reference lists the columns the import wizard recognizes for each of its three entity types: students, PACE progress, and grades. Use it to prepare a CSV before you start the wizard at Import (administrator only). The wizard itself is documented in [Importing data](/features/data/importing-data).

<Note>
The wizard supports exactly three entity types: students, PACE progress, and grades. Attendance is not an importable entity.
</Note>

## How column mapping works

You do not need to match these column names exactly. On the mapping step, the wizard reads your CSV headers and tries to match each one to a target field using a list of common variants (aliases). A header like `Student ID`, `student id`, or `ACE Number` all match the same target field, for example. Any header it cannot confidently match is left unmapped for you to assign manually, and you can also choose "Ignore this column" for headers that don&apos;t belong in Zion.

<Tip>
If a column doesn&apos;t map automatically, check it against the alias lists below. Renaming your header to the exact target field name (for example `first_name`) always works.
</Tip>

## Students template

| Field | Required | Aliases it auto-detects | Notes |
|---|---|---|---|
| First Name (`first_name`) | Yes | first name, firstname, fname, given_name, given name, forename | |
| Last Name (`last_name`) | Yes | last name, lastname, lname, surname, family_name, family name | |
| Date of Birth (`date_of_birth`) | No | date of birth, dob, birth_date, birthdate, birthday | Row is skipped if the value can&apos;t be parsed as a date |
| Gender (`gender`) | No | gender, sex | |
| Grade Level (`grade_level`) | No | grade level, grade, year, year_group, class | |
| Student Code (`student_code`) | No | student code, student id, ace number, pace id, learner_id, admission number | ACE-specific learner identifier |
| Email (`email`) | No | email, email address, e-mail, mail | |
| Phone (`phone`) | No | phone, phone number, mobile, telephone, tel, contact_number | |
| Learning Centre (`learning_centre_name`) | No | learning centre, centre, center, campus, site, location | Matched by name against your existing centres, or your org&apos;s default centre if left blank |

<Warning>
A row is skipped if First Name or Last Name is missing, or if Date of Birth is present but not a valid date. If a row would create a student and no centre can be resolved (no match and no default centre configured), that row is skipped too.
</Warning>

## PACE progress template

| Field | Required | Aliases it auto-detects | Notes |
|---|---|---|---|
| Student Code (`student_code`) | Conditional | student code, student id, ace number, pace id, learner_id | See the identifier rule below |
| Student First Name (`student_first_name`) | Conditional | student first name, first name, firstname, fname, given_name, forename | See the identifier rule below |
| Student Last Name (`student_last_name`) | Conditional | student last name, last name, lastname, lname, surname, family_name | See the identifier rule below |
| Subject (`subject`) | Yes | subject, subject_code, subject_name, course | |
| PACE Number (`pace_number`) | Yes | pace number, pace #, pace no, booklet_number, pace | Must be a whole number |
| Status (`status`) | Yes | status, progress, state, completion_status | Must be `completed`, `in_progress`, or `not_started` |
| Completed Date (`completed_date`) | No | completed date, completion_date, date_completed, finished_date | If present but unparseable, the row still imports with this field left blank |
| Score (`score`) | No | score, mark, result, percentage, percent, grade_score | Numeric, 0 to 100 |

<Note>
**Identifier rule:** every row needs a way to match it to a student. Map either Student Code, or both Student First Name and Student Last Name. Rows are matched to existing students by student code first, then by exact first and last name.
</Note>

<Warning>
A row is skipped if Subject, PACE Number, or Status is missing or invalid, for example a Status value that isn&apos;t one of the three accepted values.
</Warning>

## Grades template

| Field | Required | Aliases it auto-detects | Notes |
|---|---|---|---|
| Student Code (`student_code`) | Conditional | student code, student id, ace number, pace id, learner_id | See the identifier rule below |
| Student First Name (`student_first_name`) | Conditional | student first name, first name, firstname, fname, given_name, forename | See the identifier rule below |
| Student Last Name (`student_last_name`) | Conditional | student last name, last name, lastname, lname, surname, family_name | See the identifier rule below |
| Subject (`subject`) | Yes | subject, subject_code, subject_name, course | |
| Assessment Name (`assessment_name`) | Yes | assessment name, assessment, test_name, exam, exam_name, title | |
| Score (`score`) | Yes | score, mark, result, percentage, percent, points | Numeric, 0 or greater, capped at 999.99 |
| Max Score (`max_score`) | No | max score, maximum, out_of, total, total_marks | Defaults to 100 if left blank |
| Date (`date`) | No | date, assessment_date, test_date, exam_date, date_taken | If present but unparseable, the row still imports with this field left blank |
| Term (`term`) | No | term, semester, quarter, period, session | |

<Note>
**Identifier rule:** same as PACE progress. Map either Student Code, or both Student First Name and Student Last Name.
</Note>

<Warning>
A row is skipped if Subject, Assessment Name, or Score is missing, or if Score is negative or above 999.99.
</Warning>

## Before you import

<Steps>
  <Step title="Match one entity type per file">
    Each CSV covers one entity type only: students, PACE progress, or grades. Split mixed data into separate files before uploading.
  </Step>
  <Step title="Keep a consistent student identifier">
    Decide whether you&apos;ll match students by Student Code or by first and last name, and use that consistently across your PACE progress and grades files.
  </Step>
  <Step title="Use plain values for Status and dates">
    Status must read exactly `completed`, `in_progress`, or `not_started`. Dates in an unrecognized format import as blank rather than blocking the row.
  </Step>
  <Step title="Run the dry run before you commit">
    The wizard&apos;s dry-run step classifies every row as create, update, or skip, and lists warnings, before anything is written. Fix your CSV and re-upload if the skip count looks wrong.
  </Step>
</Steps>
