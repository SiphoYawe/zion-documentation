# Supervisor Grading Guide

As a supervisor, you&apos;re responsible for entering test scores when students complete PACE tests. This guide covers the complete workflow for test score entry, handling rejections, and managing retests.

## Your Grading Responsibilities

### After Each Test

- Enter test score into Zion
- Verify accuracy before submitting
- For failed tests, schedule retests

### Daily/Weekly

- Check for rejected scores needing re-entry
- Monitor scheduled retests
- Enter retest scores when completed

---

## Entering Test Scores

### Step 1: Access Score Entry

1. Log into Zion
2. Navigate to **Grading** in sidebar
3. The score entry form appears

### Step 2: Select Student

1. Type student name or number in search
2. Results filter as you type
3. Click to select the student

**Student Info Shown:**
- Full name
- Student number
- Grade level
- Learning centre

### Step 3: Select PACE

1. Click PACE dropdown
2. Only assigned PACEs appear
3. Select the completed PACE

<Note>
**Assigned PACEs Only:** If a PACE doesn&apos;t appear, verify it&apos;s assigned to the student in Student Management.
</Note>

### Step 4: Enter Score

1. Enter the numeric score (0-100)
2. Whole numbers only
3. System shows pass/fail immediately

**Pass Thresholds:**
- Grades 1-4: 90% required
- Grades 5-12: 80% required

### Step 5: Select Test Type

Choose the appropriate type:

| Type | When to Use |
|------|-------------|
| **PACE Test** | End-of-workbook test (most common) |
| **Self Test** | Mid-PACE check |
| **Final Test** | Comprehensive unit test |

### Step 6: Confirm Date

- Defaults to today
- Change if entering retroactively
- Cannot use future dates

### Step 7: Submit

1. Review all information
2. Click **Submit**
3. Confirmation appears

### What Happens After Submit

**For Grades 1-7:**
- Score immediately finalized
- Available for grade calculations
- Done!

**For Grades 8-12:**
- Score enters moderation queue
- Administrator will review
- Score finalizes after approval

---

## Viewing Recent Entries

### Recent Scores Tab

1. Click **Recent Scores** tab
2. See your recent entries

**Information Shown:**
- Student name
- PACE and score
- Pass/fail status
- Entry timestamp
- Moderation status (Grades 8-12)

### Status Meanings

| Status | Meaning |
|--------|---------|
| **Finalized** | Complete (Grades 1-7) |
| **Pending Moderation** | Awaiting admin review |
| **Approved** | Admin approved |
| **Rejected** | Needs re-entry |

---

## Handling Rejected Scores

When an administrator rejects your score entry, you&apos;ll need to re-enter it correctly.

### Viewing Rejections

1. Navigate to **Grading → Enter Scores**
2. Click **Rejected Scores** tab
3. See scores needing correction

### Rejection Information

Each rejected entry shows:
- Original entry details
- Rejection reason from admin
- When it was rejected

### Re-Entering a Rejected Score

<Steps>
  <Step title="Read Rejection Reason">
    Understand what was wrong
  </Step>
  <Step title="Verify Correct Information">
    Check original test paper
  </Step>
  <Step title="Enter Corrected Score">
    Submit with accurate information
  </Step>
  <Step title="Confirm Submission">
    New entry replaces rejected one
  </Step>
</Steps>

### Common Rejection Reasons

| Reason | What to Check |
|--------|---------------|
| "Wrong student" | Verify student name matches test paper |
| "Score doesn&apos;t match" | Recount from test paper |
| "Wrong PACE" | Verify PACE number on test |
| "Premature entry" | Student hasn&apos;t completed PACE |

## Managing Retests

### When a Student Fails

When a test score is below the pass threshold:

1. Student is flagged for retest
2. You can schedule the retest date
3. Student studies before retry
4. You enter retest score when complete

### Scheduling a Retest

1. Navigate to **Grading → Retests**
2. Find the failed test
3. Click **Schedule Retest**
4. Select date (must be future)
5. Confirm

### Entering Retest Scores

1. Navigate to **Grading → Retests**
2. Find scheduled retest
3. Click **Enter Score**
4. Enter the new score
5. Submit

**Important:**
- Use the retest entry form
- NOT the regular score entry
- This links to original attempt

### Retest Limits

Students have limited retest attempts:
- Default: 3 attempts total
- System tracks attempt count
- Enforced automatically

### If Student Exhausts Attempts

When max attempts reached:
- Cannot schedule more retests
- Administrator notified
- Special intervention may be needed

## Best Practices

### Score Entry

**Enter Same Day:**
- Enter scores within hours of test
- Fresh memory, fewer errors

**Double-Check:**
- Verify student name on test paper
- Confirm PACE number
- Recount score if unsure

**Don&apos;t Batch:**
- Enter one at a time
- Reduces mistakes

### Retest Scheduling

**Allow Study Time:**
- 3-5 days between attempts
- Student can review material

**Reasonable Schedule:**
- Don&apos;t overload retest days
- Balance with regular tests

### Handling Rejections

**Respond Quickly:**
- Address within 24 hours
- Don&apos;t leave pending

**Verify Carefully:**
- Re-check original test paper
- Ensure accuracy second time

## Common Issues

### PACE Not Listed

**Possible Causes:**
- Not assigned to student
- Already tested this term
- Wrong student selected

**Solution:**
- Check student&apos;s assignments
- Verify correct student

### Duplicate Error

**Cause:** Score already entered for this test

**Solution:**
- Check if already submitted
- For corrections, contact admin

### Cannot Enter Retest

**Possible Causes:**
- Max attempts reached
- Original test passed
- Retest already entered

**Solution:**
- Check attempt count
- Verify this is a failed test

## Moderation for Grades 8-12

### Understanding the Process

When you enter scores for Grades 8-12:

1. Score enters moderation queue
2. Administrator reviews
3. Admin can: Approve, Adjust, or Reject
4. You&apos;re notified of rejections

### Adjustment vs Rejection

**Adjusted:**
- Admin changed score
- Still finalized
- No action needed from you

**Rejected:**
- Returned to you
- Must re-enter correctly
- See rejection reason
