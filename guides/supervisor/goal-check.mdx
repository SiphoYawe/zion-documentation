title: "Supervisor guide: daily ritual"
description: How supervisors run the Morning Ritual and End-of-Day Check for their learning centres

---

The daily ritual is a full-screen, one-student-at-a-time way to run Goal Check for an entire learning centre in a single guided session. It writes to the same Goal Check data as the standalone entry screens, so there is no separate data store to reconcile. As a supervisor, you run the Morning Ritual to set each student&apos;s page goal, then run the End-of-Day Check to record what they actually completed.

<Note>
Administrators and monitors can also run the ritual for centres they have access to. Only administrators and supervisors can approve goals when sign-off is turned on for your organization.
</Note>

## Where to start it

Both rituals start from the **Today&apos;s Ritual** card on your dashboard. If you supervise more than one learning centre, a centre selector appears above the Start buttons so you can switch between them. Each centre tracks its own Morning Ritual and End-of-Day Check status independently, so completing one centre does not affect the others.

The card shows one of three states for each ritual:
- **Not started:** shows a Start button
- **In progress:** shows a Resume button
- **Completed today:** shows a Done state

<Tip>
If you close the tab or switch devices mid-session, opening the ritual again resumes the same session where you left off. A session started the previous day is treated as stale and does not block you from starting a fresh one today.
</Tip>

## Running the Morning Ritual

<Frame caption="Setting a page goal for each student during the Morning Ritual">
  <img src="/images/guides/supervisor/goal-check-1-am.png" alt="Morning Ritual screen showing a student's active PACEs and goal end page entry" />
</Frame>

<Steps>
  <Step title="Start or resume">
    From the Today&apos;s Ritual card, select your centre if you have more than one, then tap Start Morning Ritual, or Resume if a session is already in progress today.
  </Step>
  <Step title="Set a goal for each PACE">
    Each student appears one at a time, full-screen, with their active PACEs listed. For each PACE, the start page is filled in for you (it continues from the last page the student completed, or page 1 for a new PACE) and cannot be edited. Enter the goal end page using the number pad. It cannot exceed the PACE&apos;s total page count.
  </Step>
  <Step title="Advance to the next student">
    Tap Advance once every active PACE has a goal. If you advance with a PACE left blank, Zion asks you to confirm before moving on. A student with no goalable PACEs can still be advanced.
  </Step>
  <Step title="Use Back, Skip, or Roster as needed">
    Back returns to the previous student so you can review or change what you entered. Skip marks the current student skipped and moves to the next one. Roster opens a list of every student in the session so you can jump to any of them directly.
  </Step>
  <Step title="Finish the session">
    Before the session closes, Zion offers one more pass through any skipped students. Skip Remaining marks everyone still pending as skipped and ends the session in one step. Finish Anyway lets you close the session with students still marked skipped.
  </Step>
  <Step title="Review the summary">
    The completion screen shows how many goals were set, how many students were skipped, and how long the session took.
  </Step>
</Steps>

## Throughout the day

Between the Morning Ritual and the End-of-Day Check, use your dashboard to check on progress and support students who are struggling. See [Supervisor dashboard](/guides/supervisor/dashboard) for what the dashboard shows for your centres.

## Running the End-of-Day Check

<Frame caption="Marking progress as Hit, Missed, or Partial during the End-of-Day Check">
  <img src="/images/guides/supervisor/goal-check-2-mark.png" alt="End-of-Day Check screen showing progress marking for a student's morning goal" />
</Frame>

<Steps>
  <Step title="Start or resume">
    From the Today&apos;s Ritual card, tap Start End-of-Day Check, or Resume if a session is already in progress.
  </Step>
  <Step title="Review the morning goal">
    Each student appears with a recap of every PACE that had a morning goal.
  </Step>
  <Step title="Record the outcome">
    For each PACE, record Hit, Missed, or Partial, and enter the actual start and end pages completed. A student with no morning goal can still be advanced with a note; nothing is recorded for that PACE.
  </Step>
  <Step title="Advance, Back, Skip, or use Roster">
    These work the same way as in the Morning Ritual.
  </Step>
  <Step title="Finish and review">
    Skip Remaining, Finish Anyway, and the completion summary work the same way as the Morning Ritual, showing results recorded, students skipped, and elapsed time.
  </Step>
</Steps>

## Approving goals (if enabled)

Some organizations require sign-off on morning goals before they count as final. Whether this applies to you depends on a setting your administrator controls at the organization level.

<Tabs>
  <Tab title="If approval is off">
    Morning goals are final as soon as you enter them. There is no separate approval step.
  </Tab>
  <Tab title="If approval is on">
    Goals you set in the Morning Ritual start in a pending state. From the completion summary of that session, an administrator or supervisor can approve all of the session&apos;s pending goals in one action. A goal that is both approved and later marked complete in the End-of-Day Check is locked against casual re-marking; changing it after that point goes through the audited correction process instead of the ritual.
  </Tab>
</Tabs>

<Note>
Monitors can enter and record goals through the ritual but cannot approve them. Approval is limited to administrators and supervisors.
</Note>

## Troubleshooting

<AccordionGroup>
  <Accordion title="I closed the tab in the middle of a session">
    Reopen the Morning Ritual or End-of-Day Check for that centre. Zion resumes the same in-progress session rather than starting over.
  </Accordion>
  <Accordion title="Yesterday's session is still showing as in progress">
    A session left in progress from a previous day is treated as stale. Starting the ritual today opens a new session for that centre; it does not block you.
  </Accordion>
  <Accordion title="A student has no PACE to set a goal for">
    A PACE with no page count, or a student with no active PACE, cannot be goaled. Advance past that student; nothing is recorded for them.
  </Accordion>
  <Accordion title="I skipped a student by mistake">
    Use Back to return to a student you just passed, or open Roster to jump directly to any student in the session and correct their entry before you finish.
  </Accordion>
</AccordionGroup>
